    Linux

     Drawpile {{ VERSION }} AppImage

    Drawpile 2.2.0 is backward-compatible, so you can join sessions hosted with the previous 2.1 series. To find out how to run the AppImage, take a look at the AppImage manual and troubleshooting page. The Flatpak version is currently waiting to be integrated.

    If you run into issues, found a bug, want to suggest a feature or ask for help with something, take a look at the Help pages!

    The AppImage includes the dedicated server, which you can run by passing --server as the first argument. There's also other options to run the dedicated server, such as via Docker or by compiling from source, with some scripts provided if you want to set up a server from scratch. For more information, take a look at the dedicated server help page.

    Other Versions

    32 bit versions are available for Windows and Android.

    • Drawpile {{ VERSION }} Installer for 32 Bit Windows
    • Drawpile {{ VERSION }} Portable ZIP for 32 Bit Windows
    • Drawpile {{ VERSION }} APK for 32 Bit Android

    Note: Only use these 32 bit versions if you know that your device needs them. If not, use the regular Windows or Android versions, those are faster and more stable.

    You can get the previous Drawpile version 2.1.20 here. Support for this version will go away in the future though, so if you are having issues with 2.2, report them now so that they can be resolved before that happens. You do not need this version to join sessions hosted with Drawpile 2.1, Drawpile 2.2 is compatible with them.

    • Drawpile 2.1.20 Installer for Windows
    • Drawpile 2.1.20 Disk Image for macOS
    • Drawpile 2.1.20 Flatpak for Linux

    For systems that require it, there's a Drawpile 2.1.20 Installer for 32 Bit Windows. Do not use this unless you know you need it.

    Even older releases can be found in the Releases on GitHub. The Assets in each release contain the downloads.
